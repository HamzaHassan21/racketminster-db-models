## ERD Models

This folder contains the Conceptual and Logical Entity-Relationship Diagrams created for the RacketMinster database design.

## Conceptual ERD

File: conceptual_ERD.png
Represents the high-level structure of the system, focusing on entities, relationships, and major attributes.
Used to understand the business rules and data requirements before applying database constraints.

## Logical ERD

File: logical_ERD.png
Represents the structured database-ready model, including:

Primary keys

Foreign keys

Cardinalities

Normalised structure
This diagram is used to guide relational schema creation.

## Purpose

These diagrams were produced as part of the RacketMinster database coursework to illustrate both the conceptual and logical design stages of the system.
